# 龙岗区居家学生项目管理台 MVP

这是可本地试用的静态管理台，用于验证项目管理系统的信息架构、关键控制点、验收缺口提示和基础数据流。

## 运行方式

直接用浏览器打开 `index.html`。本版本不依赖网络、构建工具或后端服务，数据会自动保存到当前浏览器的 `localStorage`。

也可以启动本地服务端试用账号、权限、附件上传、一户一档材料库、任务派发和 JSON 文件数据库：

```bash
node server.js
```

然后访问 `http://localhost:8080`。

默认账号：

- `admin / admin123`：项目统筹
- `qc / qc123`：督导质控
- `service / service123`：一线服务人员
- `edu / edu123`：市教育局监管

## 当前功能

- 项目指标总览：正式档案、知情同意完成率、个案辅导平均进度、完整归档率。
- 验收缺口提示：自动提示知情同意、基线评估、方案审核、红码危机记录、辅导频次、服务佐证缺口。
- 家庭 / 学生档案：支持按学生、学校、家庭编号搜索，并按红黄蓝风险等级筛选。
- 增删改查与回退：支持手动建档、编辑当前档案、删除当前档案、全局搜索、回退最近操作。
- 个案详情：展示监护人、学校年级、知情同意、基线评估、服务方案审核、个案辅导进度和下一控制动作。
- 服务计划台账：展示每户服务进度、最近服务、下一任务。
- 新增服务记录：可模拟登记服务，并对未签署知情同意的心理服务进行拦截。
- 一户一档归档状态：检查知情同意、基线评估、服务方案、服务佐证、质控审核。
- 测评量表录入与报告：支持前测、中测、后测录入，并自动生成分数变化与结论。
- 危机预警工作流：按一、二、三级危机计算响应时限和下一动作。
- 低参与预警：自动识别团辅或家长参与率低于80%的家庭。
- 质控整改台账：登记整改项、识别逾期项、支持标记闭环。
- 督导台账：记录月度督导、高危专项督导与中期预审结论。
- 中期 / 终期汇编：根据当前样例数据自动生成验收摘要文本。
- 学校协同：模拟教师端反馈、学校观察、家校社医联动通知；本版不做家长端。
- 复学追踪：展示复学状态、出勤率、适应评分、主要障碍和项目级分析。
- 本地数据保存：刷新页面后保留录入数据。
- 数据导入导出：可导出 JSON 备份，也可导入同格式数据继续试用。
- 软删除与恢复：删除档案后进入“操作日志 / 已删除档案”，可恢复。
- 操作审计：记录新增、修改、删除、测评、危机、整改、通知等关键操作。
- 归档缺口清单：按家庭显示缺少哪些材料、缺少哪些服务频次。
- 人员资质与负荷：咨询师正式负荷按≤6名学生、红码≤2名校验。
- 备选准入：支持备选名单、排除、准入建档；风险等级为系统建议+人工确认。
- 验收日期：终期按 2026-09-30 口径处理，中期指标按调整后口径校验。
- 服务端试用：提供账号权限、附件上传、材料审核接口、服务端任务派发、JSON 文件数据库。

## 后续后端数据模型建议

第一阶段可先落以下核心表：

- `families`: 家庭编号、监护人信息、联系方式、服务意愿、准入状态、退出原因。
- `students`: 学生姓名、学段、学校、居家原因、居家时长、当前风险等级、复学状态。
- `consents`: 家长签署状态、学生签署状态、签署时间、文件附件、合规审核人。
- `assessments`: 量表类型、测评批次、原始分、风险结论、报告附件。
- `service_plans`: 个性化服务目标、应完成频次、责任咨询师、风险对应策略、审核状态。
- `service_records`: 服务类型、时间、对象、执行人、时长、记录正文、佐证附件、归档状态。
- `risk_events`: 危机等级、触发来源、响应时限、处置过程、转介记录、闭环状态。
- `quality_issues`: 质控问题、责任人、整改期限、整改材料、复核人、闭环时间。
- `acceptance_metrics`: 中期/终期指标、目标值、当前值、计算口径、材料来源。

## 下一阶段建议

- 接入真实名单导入和一户一档附件上传。
- 将知情同意设置为所有心理服务流程的硬拦截。
- 将红码个案、危机事件、质控整改做成独立工单流。
- 增加中期和终期验收材料快照，保障提交版本可追溯。
- 本版不做人脸签到、不做家长端；教师端仅作为轻量反馈入口，所有外部反馈先进入待审核区，再由项目组确认入档。
- 后续接入后端时，应把当前 localStorage 存储替换为数据库、账号权限、附件存储、版本号和服务端审计日志。
